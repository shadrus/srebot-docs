# Управление организациями

SREBot поддерживает **мультиорганизационную модель**: один пользователь может состоять в нескольких организациях с разными ролями в каждой. Это позволяет управлять инфраструктурой разных команд или проектов из единого аккаунта.

---

## Создание организации

После регистрации первая организация создаётся **автоматически** — пользователь сразу получает роль Владельца и может начать работу с платформой.

Чтобы создать **дополнительную** организацию:

1. Нажмите на название текущей организации в верхней навигационной панели — откроется выпадающее меню.
2. Нажмите кнопку **«+»** (или перейдите напрямую на `/dashboard/organizations/new`).
3. Заполните форму:
   - **Название организации** — произвольное имя, которое будет отображаться в навигации.
   - **Язык / Регион** — `RU` (Русский) или `EN` (English).
4. Нажмите **«Создать»**.

::: info Валюта биллинга
Валюта определяется выбранным языком: **Русский → RUB**, **English → USD**. Валюта фиксируется при создании организации и **не может быть изменена** позже.
:::

После создания вы автоматически станете **Владельцем** (Owner) новой организации и будете перенаправлены в её контекст.

---

## Членство в нескольких организациях

Пользователь может быть участником любого числа организаций, при этом роль в каждой из них назначается **независимо**.

Например, в одной организации вы можете быть Владельцем, а в другой — Наблюдателем.

Все организации, в которых вы состоите, отображаются в **выпадающем списке навигационной панели**. Каждая организация имеет собственное изолированное пространство:

- **Биллинг** — отдельный баланс и история платежей.
- **Токены ботов** — агент-токены привязаны к конкретной организации.
- **Команда** — свой список участников и приглашений.
- **Инциденты** — данные инцидентов не пересекаются между организациями.

## Переключение между организациями

Для переключения между организациями не требуется повторная авторизация — достаточно выбрать нужную организацию из списка.

1. Нажмите на **название организации** в верхней навигационной панели.
2. В выпадающем меню отобразятся все организации, в которых вы состоите. Текущая организация отмечена **галочкой** (✓).
3. Выберите нужную организацию — контекст дашборда обновится: инциденты, биллинг, команда и токены будут загружены для выбранной организации.
4. В нижней части выпадающего меню доступна кнопка **«Создать организацию»**.

::: tip Мгновенное переключение
Переключение происходит мгновенно — данные загружаются через API без перезагрузки страницы. Текущая сессия при этом не прерывается.
:::

## Контекст организации

Все данные и действия в дашборде привязаны к **текущей организации**. При переключении контекст полностью меняется:

| Раздел | Что входит в контекст |
|---|---|
| **Инциденты** | Список инцидентов, детали анализа, история tool calls |
| **Биллинг** | Баланс, тариф, история транзакций, способы пополнения |
| **Команда** | Участники, роли, ожидающие приглашения |
| **Токены ботов** | Агент-токены для подключения к Telegram, Slack, Discord |

### Роли и разрешения

Ваши права доступа определяются **членством в текущей организации**. Если вы являетесь Владельцем в одной организации и Наблюдателем в другой, то при переключении на вторую вы получите только права Наблюдателя.

Подробнее о ролях читайте в разделе [Управление командой](/guide/team-management).

### Фиксированная валюта

Валюта биллинга устанавливается **один раз** при создании организации и не может быть изменена. Если вам нужна другая валюта, создайте новую организацию с нужным языком/регионом.
