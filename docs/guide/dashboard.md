# Веб-дашборд

[Веб-дашборд SREBot](https://srebot.site360.tech) — это центр управления платформой. Здесь вы отслеживаете инциденты, управляете AI-агентами, контролируете баланс токенов и настраиваете аккаунт.

## Навигация

Интерфейс дашборда состоит из **бокового меню** (sidebar) и **верхней панели** (navbar).

### Боковое меню

В боковом меню находятся ссылки на все разделы дашборда:

| Раздел    | Путь                   | Описание                                       |
| --------- | ---------------------- | ---------------------------------------------- |
| Обзор     | `/dashboard`           | Статистика использования и последние инциденты |
| Инциденты | `/dashboard/incidents` | Список всех инцидентов с фильтрацией           |
| Боты      | `/dashboard/bots`      | Управление токенами AI-агентов                 |
| Биллинг   | `/dashboard/billing`   | Баланс, тарифы, история транзакций             |
| Настройки | `/dashboard/settings`  | Пароль и профиль организации                   |

::: info Видимость разделов зависит от роли
Разделы **Биллинг** и **Команда** отображаются только для пользователей с ролью **Владелец** или **Администратор**. Участники и Наблюдатели видят только Обзор, Инциденты и Настройки.
:::

### Переключатель организаций

В верхней панели находится **выпадающий список организаций**. Если вы состоите в нескольких организациях, нажмите на название текущей организации, чтобы переключиться на другую. Все данные дашборда (инциденты, баланс, боты) привязаны к выбранной организации.

## Обзор (`/dashboard`)

Главная страница дашборда предоставляет быстрый срез активности вашей организации.

### Карточки статистики

В верхней части страницы расположены три карточки с ключевыми метриками использования:

| Карточка             | Описание                                              |
| -------------------- | ----------------------------------------------------- |
| **Всего инцидентов** | Общее количество обработанных инцидентов за всё время |
| **Промпт-токены**    | Количество входных токенов, отправленных в LLM        |
| **Токены генерации** | Количество выходных токенов, сгенерированных LLM      |

На каждой карточке отображается **процент тренда за неделю** — зелёная стрелка при росте и красная при снижении — чтобы вы могли оценить динамику потребления.

### Последние инциденты

Под карточками находится таблица **5 последних инцидентов** с краткой информацией:

| Столбец         | Описание                                                       |
| --------------- | -------------------------------------------------------------- |
| **Алерт**       | Название алерта, вызвавшего инцидент                           |
| **Бот**         | Имя AI-агента, обработавшего инцидент                          |
| **Статус**      | Цветной индикатор: 🟢 `completed`, 🟡 `analyzing`, 🔴 `failed` |
| **Время**       | Дата и время обнаружения                                       |
| **Критичность** | Уровень серьёзности: `high`, `medium`, `low`                   |

Нажмите на строку инцидента, чтобы перейти к его детальной карточке.

## Управление ботами (`/dashboard/bots`)

::: new
Раздел **Боты** — это новое место для управления токенами AI-агентов вашей организации. Здесь вы создаете, просматриваете и отзываете токены, которые боты используют для подключения к платформе SREBot.
:::

### Список токенов

На странице отображается таблица всех токенов агентов в организации:

| Столбец           | Описание                                     |
| ----------------- | -------------------------------------------- |
| **Имя**           | Название бота, указанное при создании токена |
| **Дата создания** | Когда был сгенерирован токен                 |
| **Статус**        | `Активен` или `Отозван`                      |

### Создание нового токена

1. Нажмите кнопку **«Создать токен»** в правом верхнем углу.
2. В модальном окне введите **имя бота** (например, `prod-bot-1`).
3. Нажмите **«Создать»**.
4. Система сгенерирует секретный токен и отобразит его в модальном окне.

::: danger Токен показывается только один раз
Сразу после создания скопируйте токен и сохраните его в безопасном месте (например, в менеджере секретов). После закрытия модального окна **увидеть токен повторно невозможно**. Если вы потеряете токен, его придется отозвать и создать новый.
:::

Для удобства рядом с токеном расположена кнопка **копирования в буфер обмена**.

### Отзыв токена

Чтобы отозвать токен:

1. Нажмите на иконку корзины в строке нужного токена.
2. В диалоге подтверждения нажмите **«Отозвать»**.

После отзыва бот немедленно потеряет доступ к платформе. Действие необратимо — если нужно восстановить доступ, создайте новый токен.

### Лимит токенов

::: warning Максимум 10 токенов
Каждая организация может иметь не более **10 активных токенов**. При попытке создать одиннадцатый система покажет ошибку. Если вам нужно больше токенов, сначала отзовите неиспользуемые.
:::

## Список инцидентов (`/dashboard/incidents`)

Страница со списком всех инцидентов организации с поддержкой поиска, фильтрации и пагинации.

### Фильтры

Над таблицей расположены элементы управления:

| Фильтр             | Описание                                |
| ------------------ | --------------------------------------- |
| **Поиск**          | Полнотекстовый поиск по названию алерта |
| **Дата начала**    | Фильтр по начальной дате обнаружения    |
| **Дата окончания** | Фильтр по конечной дате обнаружения     |

### Таблица инцидентов

Инциденты отображаются в постраничной таблице (по **20 записей на страницу**) со следующими столбцами:

| Столбец         | Описание                                                   |
| --------------- | ---------------------------------------------------------- |
| **Алерт**       | Название алерта (например, `HighCpuUsage`)                 |
| **Бот**         | Имя агента, обработавшего инцидент                         |
| **Статус**      | Цветной бейдж: 🟡 `analyzing`, 🟢 `completed`, 🔴 `failed` |
| **Время**       | Дата и время обнаружения инцидента                         |
| **Критичность** | Уровень серьёзности (`high` / `medium` / `low`)            |
| **Действия**    | Кнопка перехода к детальной карточке                       |

Для навигации между страницами используйте элементы пагинации внизу таблицы.

## Детальная карточка инцидента (`/dashboard/incidents/:id`)

Страница детального просмотра инцидента — ключевой инструмент для анализа работы AI-агента. Это позволяет проверить корректность суждений бота (hallucination check) и переиспользовать его PromQL-запросы для создания новых дашбордов в Grafana.

### Анализ первопричины (Root Cause Analysis)

Основной блок страницы содержит **итоговый вывод LLM** об инциденте, отрендеренный в формате Markdown. В этом блоке бот описывает:

- что произошло;
- вероятную причину сбоя;
- рекомендации по устранению.

### Уточняющие вопросы (Follow-up Questions)

Если к инциденту задавались уточняющие вопросы через чат, они отображаются ниже блока анализа в виде пар **«Вопрос → Ответ»**. Для каждого follow-up показывается:

- текст вопроса пользователя;
- ответ бота;
- **детализация токенов** — сколько входных и выходных токенов было затрачено на конкретный ответ.

### Шаги анализа (Analysis Steps)

Блок с раскрывающимися секциями, показывающий **мыслительный процесс LLM** — какие инструменты и запросы бот использовал при анализе инцидента.

Каждый шаг содержит:

| Элемент               | Описание                                                                       |
| --------------------- | ------------------------------------------------------------------------------ |
| **Фаза**              | Название этапа (например, `PromQL Query`, `Elasticsearch Search`)              |
| **Аргументы запроса** | Параметры, переданные в инструмент (например, PromQL-выражение или DSL-запрос) |
| **Ответ**             | Сокращённый payload, который вернул инструмент                                 |

Нажмите на заголовок шага, чтобы развернуть или свернуть детали.

::: tip Переиспользование запросов
Если бот выполнил удачный PromQL-запрос, вы можете скопировать его из шагов анализа и использовать для создания дашборда в Grafana или настройки алерт-правил.
:::

### Боковая панель с метаданными

Справа от основного контента расположена панель с метаинформацией об инциденте:

| Поле                | Описание                                        |
| ------------------- | ----------------------------------------------- |
| **Длительность**    | Время, затраченное на анализ                    |
| **Обнаружен**       | Дата и время обнаружения инцидента              |
| **Входные токены**  | Количество токенов, отправленных в LLM          |
| **Выходные токены** | Количество токенов, сгенерированных LLM         |
| **Бот**             | Имя агента, обработавшего инцидент              |
| **ID инцидента**    | Уникальный идентификатор                        |
| **Критичность**     | Уровень серьёзности (`high` / `medium` / `low`) |

## Биллинг и токены (`/dashboard/billing`)

SREBot использует **token-based** систему биллинга для оплаты вызовов LLM. Вся информация о расходах и пополнениях сосредоточена на этой странице.

### Текущий баланс

В верхней части страницы отображается **текущий баланс токенов** организации. Под балансом находится **мини-столбчатая диаграмма за последние 14 дней**, где:

- **зелёные столбцы** — пополнения (кредиты);
- **красные столбцы** — списания за использование (дебеты).

Диаграмма позволяет быстро оценить динамику расходования средств.

### Текущий тариф

Рядом с балансом отображается карточка **активного тарифного плана**:

| Параметр               | Описание                                                                                  |
| ---------------------- | ----------------------------------------------------------------------------------------- |
| **Цена за 1M токенов** | Стоимость одного миллиона входных токенов                                                 |
| **Множитель выхода**   | Коэффициент, на который умножается цена для выходных токенов (output обычно дороже input) |
| **Валюта**             | Валюта расчёта — **RUB** или **USD**                                                      |

::: info Выбор валюты
Валюта организации определяется **при её создании** на основе выбранного языка интерфейса и не может быть изменена позже.
:::

### Пополнение баланса

1. Нажмите кнопку **«Пополнить»** рядом с балансом.
2. В модальном окне введите **количество токенов** для пополнения.
   - Минимум: **100 000 токенов**
   - Максимум: **100 000 000 токенов**
3. Система автоматически рассчитает **ориентировочную стоимость** в валюте вашей организации.
4. Нажмите **«Оплатить»** — вы будете перенаправлены на платёжный шлюз.

После успешной оплаты баланс обновится автоматически.

### История транзакций

В нижней части страницы находится таблица всех операций с балансом с возможностью фильтрации:

| Фильтр             | Описание                           |
| ------------------ | ---------------------------------- |
| **Тип транзакции** | Фильтр по категории (см. ниже)     |
| **Диапазон дат**   | Фильтр по дате совершения операции |

#### Типы транзакций

| Тип        | Описание                                           |
| ---------- | -------------------------------------------------- |
| **Usage**  | Списание за анализ инцидента или уточняющий вопрос |
| **Top-up** | Пополнение баланса через платёжный шлюз            |
| **Gift**   | Подарочные токены, начисленные администратором     |
| **Refund** | Возврат средств (например, при ошибочном списании) |

Транзакции типа **Usage** содержат ссылку на связанный инцидент — нажмите на неё, чтобы перейти к детальной карточке и посмотреть, за что именно были списаны токены.

### Защита от отрицательного баланса

::: danger Блокировка анализа
Если баланс падает **ниже нуля**, SREBot **прекращает анализ новых оповещений**. Вместо этого в чат (Telegram/Slack/Discord) приходит уведомление о необходимости пополнить баланс. Анализ возобновляется автоматически после пополнения.
:::

## Аккаунт и безопасность (`/dashboard/settings`)

На странице настроек вы управляете параметрами своего профиля и контекстом организации.

### Смена пароля

Для смены пароля:

1. Введите текущий пароль в поле **«Текущий пароль»**.
2. Введите новый пароль в поле **«Новый пароль»** (минимум 8 символов).
3. Подтвердите новый пароль в поле **«Подтвердите пароль»**.
4. Нажмите **«Сохранить»**.

### Контекст организации

На странице настроек отображается информация о текущей организации:

- **Название организации**
- **Ваша роль** в этой организации (Owner / Admin / Member / Viewer)

Для управления участниками организации перейдите в раздел **«Команда»** (доступно только для Владельцев и Администраторов). Подробнее об этом читайте в разделе [Управление командой](/guide/team-management).
