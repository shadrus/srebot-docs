# Интерактивный Веб-дашборд

[Веб-дашборд SREBot](/#) — это центр управления вашей обсервабилити. В нём вы не только управляете настройками аккаунта, но и можете детально просматривать работу AI-агентов по каждому инциденту.

## Детализация инцидентов (Incident Analysis View)

Одной из самых главных особенностей SREBot является прозрачность его работы («thought process»). На странице инцидентов выводится сводка всех произошедших сбоев с цветовой индикацией тяжести.

При клике на инцидент система переводит вас на **Детальную карточку инцидента**, где отображается:
- **Статус анализа:** Выполняется процесс сбора (`analyzing`), завершен (`completed`) или произошла ошибка (`failed`).
- **Текстовое описание проблемы:** Итоговый вывод об инциденте.
- **Инструменты LLM (Tool Calls):** История того, какие конкретно запросы (PromQL или поиск в Elasticsearch DSL) совершал бот. 

Это позволяет аналитикам проверить корректность суждений бота (hallucination check) и переиспользовать его PromQL-запросы для создания новых дашбордов в Grafana.

## Биллинг и токены

SREBot имеет гибкую **token-based** биллинг систему для оплаты API LLM.

- **Тарифы:** Стоимость рассчитывается за каждый 1 миллион входных (`input`) и выходных (`output`) токенов. Эта стоимость привязана к активному тарифному плану (Billing Tariff).
- **Списание логика (Usage Allocation):** Списание происходит в долларах (USD) и пересчитывается автоматически после каждого завершенного инцидента.
- **Просмотр истории:** На вкладке Billing находится визуальный график потребления баланса и полная история денежных транзакций (Пополнения, Списания заUsage и прочее).
- **Пополнение:** Поддерживаются различные способы оплаты, включая интеграцию с Robokassa и Gift карточки.

::: danger Блокировка работы
Если ваш баланс падает ниже `0.0`, SREBot перестает анализировать новые оповещения. Вместо этого в Telegram будет приходить уведомление о необходимости пополнения баланса.
:::

## Управление аккаунтом и безопасность

- **Аутентификация:** Имеется безопасная авторизация и система сброса пароля (через email).
- Управление профилем пользователя и привязанной организацией (командой).
