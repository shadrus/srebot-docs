# Настройка интеграции с Telegram

Интеграция SREBot с Telegram позволяет боту читать сообщения в вашем групповом чате, находить в них алерты от Alertmanager и автоматически отвечать готовым RCA-отчётом.

---

## 1. Создание бота через @BotFather

1. Откройте Telegram и найдите **@BotFather** — официального бота для управления ботами.
2. Отправьте команду `/newbot`.
3. BotFather попросит ввести **отображаемое имя** бота (например: `SREBot — Acme Prod`).
4. Затем введите **username** бота — он должен заканчиваться на `bot` (например: `acme_sre_bot`).
5. BotFather создаст бота и вернёт **токен** вида:

   ```
   123456789:AAFhd-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

   Сохраните этот токен — он понадобится на шаге настройки.

> [!IMPORTANT]
> **Один инстанс на один токен:** Telegram допускает только **одно** активное соединение (long-polling) для конкретного токена бота.
>
> - Если запустить две копии SREBot с одним и тем же токеном, они будут «перехватывать» сообщения друг у друга, что приведет к непредсказуемой работе и потере алертов.
> - Если вам нужно мониторить разные кластеры или окружения независимо, **создайте уникального бота для каждого инстанса SREBot** и пригласите их всех в ваш инцидент-чат.

> [!TIP]
> Username бота будет виден всем участникам чата. Рекомендуем использовать что-то понятное команде, например `acme_sre_bot`.

---

## 2. Настройка прав бота (Group Privacy)

По умолчанию Telegram-боты в группах получают **только сообщения, начинающиеся с `/`** (команды) или упоминания бота. Чтобы SREBot мог читать все сообщения (включая алерты от Alertmanager), нужно отключить режим Group Privacy.

1. В диалоге с **@BotFather** отправьте `/mybots`.
2. Выберите вашего бота из списка.
3. Нажмите **Bot Settings → Group Privacy**.
4. Нажмите **Turn off**.
5. Убедитесь, что статус изменился на `Group Privacy is DISABLED`.

> [!IMPORTANT]
> Без этого шага бот **не будет видеть** сообщения Alertmanager и никогда не запустит анализ.

---

## 3. Получение Chat ID группового чата

SREBot нужно сообщить ID конкретного чата, за которым он будет следить. Чтобы его узнать:

**Способ 1 — через @userinfobot:**

1. Добавьте [@userinfobot](https://t.me/userinfobot) во временный чат или напишите ему в личку.
2. Перешлите в него любое сообщение из вашего инцидент-чата.
3. Бот ответит с данными отправителя и `Forwarded from chat` с числовым ID.

**Способ 2 — через Telegram API:**

1. Добавьте вашего бота в нужный чат.
2. Отправьте в чат любое сообщение.
3. Откройте в браузере:
   ```
   https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
   ```
4. В ответе найдите поле `"chat": {"id": -100123456789, ...}`.

> [!NOTE]
> ID групповых чатов всегда **отрицательный** и начинается с `-100`. Например: `-1001234567890`.

---

## 4. Добавление бота в чат

1. Откройте нужный инцидент-чат в Telegram.
2. Нажмите на название чата → **Добавить участника**.
3. Найдите бота по его username и добавьте.
4. Убедитесь, что у бота есть права **читать сообщения** — для обычных групп это происходит автоматически после отключения Group Privacy (шаг 2).

> [!TIP]
> Если чат является **супергруппой** или **каналом** — добавьте бота как **администратора** с правом чтения сообщений. Это требуется для чтения сообщений из каналов (posted через Alertmanager webhook).

---

## 5. Настройка бота (Env)

После сбора всех данных добавьте их в ваш файл `.env` или в `values.yaml` Helm-чарта:

```bash
# Токен бота от BotFather
TELEGRAM_BOT_TOKEN=123456789:AAFhd-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# ID вашего группового чата (отрицательное число)
TELEGRAM_CHANNEL_ID=-1001234567890
```

Или в `values.yaml`:

```yaml
config:
  telegramBotToken: "123456789:AAFhd-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
  telegramChatId: "-1001234567890"
```

---

## Как это работает после настройки

После запуска SREBot подключается к Telegram через **long-polling** и ждёт новых сообщений в указанном чате. При появлении алерта от Alertmanager бот:

1. Парсит сообщение и извлекает метаданные алерта.
2. Проверяет дедупликацию — если такой алерт уже обрабатывается, пропускает его.
3. Отправляет в тред **reply-заглушку** `🔍 Analyzing...` на оригинальное сообщение.
4. Запускает AI-анализ и заменяет заглушку готовым RCA-отчётом.
