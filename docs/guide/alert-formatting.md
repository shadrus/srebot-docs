# Форматирование алертов

Для того чтобы SREBot быстро и надежно распознавал ваши уведомления без лишних затрат на нейросеть, рекомендуется следовать определенным стандартам при настройке шаблонов в Alertmanager.

Бот поддерживает **гибридный парсинг**: сначала пробует быстрые регулярные выражения (локально), а если они не срабатывают — использует интеллектуальный парсинг через LLM (на бекенде).

---

## 🚀 Рекомендуемые шаблоны

### 1. Стандартный (самый быстрый)
Этот формат лучше всего подходит для больших объемов данных, так как обрабатывается мгновенно простыми регулярками.

**Пример:**
```text
🚨 Alerts Firing:

Labels:
 - alertname = CPUUsageHigh
 - cluster = yandex-production
 - severity = critical
 - namespace = default
Annotations:
 - summary = CPU usage is above 90%
Source: http://prometheus/graph
```

### 2. Markdown/Bold (для Slack/Telegram)
Если вы используете жирный шрифт для ключей, бот поймет и такой формат.

**Пример:**
```text
🔥 [FIRING:1] KubePodNotReady
*Alert:* Pod has been non-ready for more than 15 minutes.
*Details:*
 • *alertname:* `KubePodNotReady`
 • *cluster:* `yandex-production`
 • *namespace:* `canton`
 • *severity:* `warning`

*Annotations:*
 • *summary:* Pod validator-app is failing
Source: http://alertmanager/details
```

## 🔍 Обязательные поля

Для корректной работы анализа боту критически важны следующие метки (labels):

| Метка | Описание |
| :--- | :--- |
| `alertname` | Идентификатор проблемы (по нему ищется история). |
| `cluster` | Название кластера. Должно совпадать с ключом в `config.yml` для работы инструментов (Prometheus/Elastic). |
| `severity` | Важность (critical, warning, info). |
| `namespace` | (Желательно) Помогает сузить поиск логов и метрик до конкретного окружения. |

---

## 🛠 Умный парсинг (Smart Fallback)

Если ваш шаблон сильно отличается от стандартных (например, вы присылаете просто текст «У нас упал прод в кластере X»), бот отправит это сообщение на **интеллектуальный парсинг** на бекенд.

> [!NOTE]
> Интеллектуальный парсинг тратит токены вашей организации и логируется на бекенде. Мы рекомендуем использовать стандартные шаблоны для автоматических алертов и оставлять Smart-режим для внештатных ситуаций или ручных сообщений.

---

## 💡 Советы по настройке
1. **Используйте `cluster`**: Всегда добавляйте метку `cluster`. Если бот не найдет в алерте кластер, он не сможет вызвать инструменты диагностики (MCP), так как не будет знать, к какому Prometheus обращаться.
2. **Разделяйте Firing и Resolved**: Используйте четкие заголовки (или эмодзи 🚨/🔥 для Firing и ✅ для Resolved) в начале сообщения.
3. **Source URL**: Ссылка `Source:` в конце сообщения позволяет боту (и вам) быстро перейти в интерфейс Alertmanager/Prometheus.
