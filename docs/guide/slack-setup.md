# Настройка интеграции со Slack

Интеграция SREBot со Slack позволяет боту анализировать сообщения в каналах, искать в них алерты от Alertmanager и проводить их автоматический анализ с помощью ИИ.

Бот работает в режиме **Socket Mode**, что избавляет от необходимости настраивать публичные Webhooks и позволяет запускать бота даже за NAT или в закрытом контуре.

---

## 1. Регистрация приложения в Slack

1. Перейдите на [Slack API: Your Apps](https://api.slack.com/apps).
2. Нажмите **Create New App** -> **From scratch**.
3. Назовите приложение (например, `SREBot`) и выберите свой Workspace.

## 2. Настройка Socket Mode

1. В боковом меню выберите **Settings -> Socket Mode**.
2. Включите переключатель **Enable Socket Mode**.
3. Slack предложит создать **App-Level Token**. 
   - Назовите его (например, `srebot-socket`).
   - Добавьте область (scope) `connections:write`.
   - **Сохраните этот токен** – он начнется на `xapp-...`. В конфиге бота это `SLACK_APP_TOKEN`.

## 3. Настройка разрешений (Scopes)

1. Перейдите в **Features -> OAuth & Permissions**.
2. Прокрутите до раздела **Scopes** -> **Bot Token Scopes**.
3. Добавйте следующие разрешения:
   - `channels:history` — чтобы бот видел сообщения в публичных каналах.
   - `groups:history` — чтобы бот видел сообщения в приватных каналах.
   - `chat:write` — чтобы бот мог отвечать в каналах.
   - `app_mention:read` — чтобы бот реагировал на упоминания через `@`.

4. Прокрутите вверх и нажмите **Install to Workspace**. После подтверждения вы получите **Bot User OAuth Token** (начинается на `xoxb-...`).
   - Это `SLACK_BOT_TOKEN` в конфиге бота.

## 4. Подписка на события (Event Subscriptions)

1. Перейдите в **Features -> Event Subscriptions**.
2. Включите **Enable Events**.
3. В разделе **Subscribe to bot events** добавьте:
   - `message.channels`
   - `message.groups` (если планируете использовать в приватных каналах)
   - `app_mention`
4. Нажмите **Save Changes**.

## 5. Настройка бота (Env)

Добавьте полученные токены и ID канала в ваш файл `.env` или настройки Helm:

```bash
# Токен бота (xoxb-...)
SLACK_BOT_TOKEN=xoxb-your-bot-token
# Токен приложения (xapp-...)
SLACK_APP_TOKEN=xapp-your-app-token
# ID канала, где бот должен слушать алерты
SLACK_CHANNEL_ID=C0123456789
```

> [!TIP]
> Чтобы узнать ID канала, нажмите на название канала в Slack -> Settings -> в самом низу будет "Channel ID".

## 6. Приглашение бота в канал

Бот не будет видеть сообщения, пока вы его не добавите в нужный канал:
1. Зайдите в нужный канал в Slack.
2. Напишите `/invite @имя_вашего_бота`.

---

## Как это работает

- **Автоматический анализ**: Если бот добавлен в канал и имеет права `channels:history`, он автоматически подхватывает любые сообщения, похожие на алерты Alertmanager, и начинает их анализ.
- **Упоминания**: Вы можете отправить алерт и тегнуть бота (`@SREBot`), чтобы он принудительно проанализировал сообщение.
- **Тред (Ветки)**: Бот присылает результат анализа в тот же канал. Если это "Firing" алерт, он сначала создает сообщение-заглушку "Analyzing...", а затем редактирует его, заменяя на полноценный отчет.
