# Настройка интеграции с Discord

Интеграция SREBot с Discord позволяет боту анализировать сообщения в каналах, искать в них алерты от Alertmanager и проводить их автоматический анализ с помощью ИИ.

---

## 1. Регистрация приложения в Discord

1. Перейдите на [Discord Developer Portal](https://discord.com/developers/applications).
2. Нажмите кнопку **New Application**.
3. Введите название (например, `SREBot`) и подтвердите создание.

## 2. Создание бота и получение токена

1. В боковом меню выберите раздел **Bot**.
2. Нажмите **Reset Token** (или **Copy Token**, если он уже виден), чтобы получить токен.
   - **Сохраните этот токен!** В конфиге бота это `DISCORD_BOT_TOKEN`.
3. В этом же разделе прокрутите вниз до **Privileged Gateway Intents**.
4. **ОБЯЗАТЕЛЬНО** включите переключатель **Message Content Intent**. Без этого бот не сможет читать тексты алертов.
5. Нажмите **Save Changes**.

## 3. Приглашение бота на сервер

1. Перейдите в раздел **OAuth2 -> URL Generator**.
2. В списке **Scopes** выберите `bot`.
3. В появившемся списке **Bot Permissions** выберите следующие права:
   - `Read Messages / View Channels`
   - `Send Messages`
   - `Embed Links` (рекомендуется)
   - `Read Message History`
4. Скопируйте сгенерированную ссылку внизу страницы и откройте её в браузере.
5. Выберите свой сервер и подтвердите добавление бота.

## 4. Получение ID канала

1. В Discord перейдите в **User Settings -> Advanced**.
2. Включите **Developer Mode**.
3. Щелкните правой кнопкой мыши по каналу, в котором бот должен слушать алерты, и выберите **Copy Channel ID**.
   - Это числовое значение будет `DISCORD_CHANNEL_ID` в конфиге бота.

## 5. Настройка бота (Env)

Добавьте полученные данные в ваш файл `.env` или настройки Helm:

```bash
# Токен бота из Discord Developer Portal
DISCORD_BOT_TOKEN=MTAx...your.token...
# ID канала (Snowflake ID)
DISCORD_CHANNEL_ID=123456789012345678
```

---

## Как это работает

- **Автоматический анализ**: Если бот добавлен в канал и имеет права на чтение сообщений, он автоматически подхватывает любые сообщения, похожие на алерты Alertmanager, и начинает их анализ.
- **Поддержка тредов**: Если вы используете форумы или ветки (Threads) в Discord, бот будет отвечать именно в той ветке, где был опубликован алерт.
- **Индикация процесса**: При получении алерта бот сначала отправляет сообщение `🔍 Analyzing...`, а затем редактирует его, заменяя на полноценный RCA-отчёт.
- **Обновление статуса**: Если алерт переходит в статус `Resolved`, бот пришлет подтверждение ответом на оригинальное сообщение.

---

## Резюме: необходимые права бота

Для корректной работы SREBot в Discord должны быть включены следующие опции:

**Developer Portal (Bot раздел):**
- ✅ **Message Content Intent** (раздел Privileged Gateway Intents)

**Developer Portal (URL Generator):**
- ✅ **View Channels** (General Permissions)
- ✅ **Send Messages** (Text Permissions)
- ✅ **Read Message History** (Text Permissions)

**Настройки Discord сервера (для приватных каналов):**
Если канал приватный, добавьте бота в исключения и разрешите:
- ✅ **View Channel**
- ✅ **Send Messages**
- ✅ **Read Message History**

> [!IMPORTANT]
> Убедитесь, что у бота есть роль с правом просмотра истории сообщений (`Read Message History`), иначе он не сможет корректно связывать Resolved-уведомления с исходными алертами.
