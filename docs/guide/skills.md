# Скилы (Bot Skills)

Скилы — это настраиваемые текстовые инструкции (в формате Markdown), позволяющие адаптировать поведение LLM-агента SREBot под специфику инфраструктуры вашей организации без внесения изменений в код бота.

## Зачем нужны скилы?

По умолчанию SREBot использует общие алгоритмы анализа логов и метрик, однако в реальных проектах инфраструктура часто имеет уникальные особенности:
- Специфические форматы логирования (например, имя приложения находится во вложенном поле `json.appname` или `json.app_name`, а не `app_name`).
- Нестандартное размещение метрик или кастомные PromQL-запросы.
- Уникальные ссылки на внутренние системы логирования (Kibana, Loki) или документацию.

Скилы позволяют передать эти знания модели при расследовании инцидентов.

## Создание скила

Чтобы создать скил, перейдите в раздел **Скилы** в боковом меню дашборда и нажмите кнопку **Создать скил**.

Форма создания скила состоит из следующих полей:

### 1. Метаданные
- **Название (Name):** Понятное имя скила (например, `Правила парсинга логов worker-billing`).
- **Описание (Description):** Краткое описание назначения скила.

### 2. Область видимости (Scope)
Определяет, для каких именно агентов будут применены инструкции:
- **Организация (Organization):** Инструкции будут применяться для всех ботов внутри вашей организации.
- **Конкретный бот (Specific Bot):** Инструкции привязываются только к выбранным ботам.

### 3. Правила триггеров (Trigger Rules)
Позволяют автоматически активировать скил только для определенных алертов. Вы можете указать набор правил в формате `Ключ: Регулярное выражение`:
- Например: `namespace: production`, `alertname: OOMKilled`.
- Если правила триггеров не заданы, скил будет являться **общим (catch-all)** и применяться ко всем инцидентам в выбранной области видимости.

### 4. Инструкция (Markdown Instructions)
Свободный текст в формате Markdown, который описывает правила и контекст. Этот текст динамически подмешивается в системный промпт LLM-модели.

## Принцип работы

При получении нового алерта бэкенд SREBot автоматически выполняет следующие шаги:
1. **Извлечение метаданных:** Извлекаются все метки (labels) и аннотации (annotations) из входящего алерта.
2. **Матчинг триггеров:** На основе регулярных выражений проверяются правила триггеров для всех активных скилов организации.
3. **Сортировка по приоритету:** Все подошедшие скилы сортируются по области видимости: сначала **Глобальные (Global)**, затем **Организационные (Organization)**, и в последнюю очередь **Локальные (Bot-specific)**. Инструкции конкретного бота имеют наивысший приоритет благодаря расположению в конце системного промпта.
4. **Инъекция в промпт:** Сформированный блок инструкций добавляется в системное сообщение LLM перед запуском цикла расследования.
