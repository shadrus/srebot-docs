# Interactive Web Dashboard

The [SREBot Web Dashboard](https://srebot.site360.tech) is your platform's central control panel. Here you track incidents, manage AI agents, monitor token balances, and configure your account.

## Navigation

The dashboard interface consists of a **sidebar menu** and a **top navigation bar**.

### Sidebar Menu

The sidebar contains links to all dashboard sections:

| Section   | Path                   | Description                           |
| --------- | ---------------------- | ------------------------------------- |
| Overview  | `/dashboard`           | Usage statistics and recent incidents |
| Incidents | `/dashboard/incidents` | Full incident list with filtering     |
| Bots      | `/dashboard/bots`      | AI agent token management             |
| Billing   | `/dashboard/billing`   | Balance, tariffs, transaction history |
| Settings  | `/dashboard/settings`  | Password and organization profile     |

::: info Section visibility depends on role
The **Billing** and **Team** sections are visible only to users with the **Owner** or **Admin** role. Members and Viewers see only Overview, Incidents, and Settings.
:::

### Organization Switcher

The top navigation bar contains an **organization dropdown**. If you belong to multiple organizations, click the current organization name to switch. All dashboard data (incidents, balance, bots) is scoped to the selected organization.

## Overview (`/dashboard`)

The main dashboard page provides a quick snapshot of your organization's activity.

### Statistics Cards

At the top of the page are three cards showing key usage metrics:

| Card                  | Description                                     |
| --------------------- | ----------------------------------------------- |
| **Total Incidents**   | Total number of processed incidents of all time |
| **Prompt Tokens**     | Number of input tokens sent to the LLM          |
| **Generation Tokens** | Number of output tokens generated by the LLM    |

Each card displays a **weekly trend percentage** — a green arrow for growth and a red arrow for decline — so you can assess consumption dynamics at a glance.

### Recent Incidents

Below the cards is a table of the **5 most recent incidents** with summary information:

| Column       | Description                                                  |
| ------------ | ------------------------------------------------------------ |
| **Alert**    | Name of the alert that triggered the incident                |
| **Bot**      | Name of the AI agent that processed the incident             |
| **Status**   | Color indicator: 🟢 `completed`, 🟡 `analyzing`, 🔴 `failed` |
| **Time**     | Date and time of detection                                   |
| **Severity** | Severity level: `high`, `medium`, `low`                      |

Click on an incident row to navigate to its detail page.

## Bot Management (`/dashboard/bots`)

::: new
The **Bots** section is the new home for managing your organization's AI agent tokens. Here you create, view, and revoke the tokens that bots use to connect to the SREBot platform.
:::

### Token List

The page displays a table of all agent tokens in the organization:

| Column      | Description                                |
| ----------- | ------------------------------------------ |
| **Name**    | Bot name specified when creating the token |
| **Created** | When the token was generated               |
| **Status**  | `Active` or `Revoked`                      |

### Creating a New Token

1. Click the **"Create Token"** button in the top-right corner.
2. In the modal dialog, enter a **bot name** (e.g., `prod-bot-1`).
3. Click **"Create"**.
4. The system generates a secret token and displays it in the modal.

::: danger Token is shown only once
Immediately after creation, copy the token and store it in a secure location (e.g., a secrets manager). After closing the modal, **the token cannot be viewed again**. If you lose the token, you must revoke it and create a new one.
:::

For convenience, a **copy-to-clipboard** button is placed next to the token.

### Revoking a Token

To revoke a token:

1. Click the trash icon on the token's row.
2. In the confirmation dialog, click **"Revoke"**.

Once revoked, the bot immediately loses access to the platform. This action is irreversible — to restore access, create a new token.

### Token Limit

::: warning Maximum 10 tokens
Each organization can have at most **10 active tokens**. Attempting to create an eleventh will show an error. If you need more tokens, revoke unused ones first.
:::

## Incident List (`/dashboard/incidents`)

A paginated list of all organization incidents with search, filtering, and pagination support.

### Filters

Above the table are filter controls:

| Filter         | Description                       |
| -------------- | --------------------------------- |
| **Search**     | Full-text search by alert name    |
| **Start Date** | Filter by earliest detection date |
| **End Date**   | Filter by latest detection date   |

### Incident Table

Incidents are displayed in a paginated table (**20 records per page**) with the following columns:

| Column       | Description                                              |
| ------------ | -------------------------------------------------------- |
| **Alert**    | Alert name (e.g., `HighCpuUsage`)                        |
| **Bot**      | Name of the agent that processed the incident            |
| **Status**   | Color badge: 🟡 `analyzing`, 🟢 `completed`, 🔴 `failed` |
| **Time**     | Date and time the incident was detected                  |
| **Severity** | Severity level (`high` / `medium` / `low`)               |
| **Actions**  | Button to navigate to the incident detail page           |

Use the pagination controls at the bottom of the table to navigate between pages.

## Incident Detail (`/dashboard/incidents/:id`)

The incident detail page is the key tool for analyzing the AI agent's work. It allows you to verify the bot's reasoning (hallucination check) and reuse its PromQL queries for building new Grafana dashboards.

### Root Cause Analysis

The main block of the page contains the **final LLM output** for the incident, rendered as Markdown. In this block, the bot describes:

- What happened;
- The probable cause of the failure;
- Recommendations for remediation.

### Follow-up Questions

If follow-up questions were asked about the incident via chat, they appear below the analysis block as **"Question → Answer"** pairs. For each follow-up, the following is displayed:

- The user's question text;
- The bot's response;
- **Token breakdown** — how many input and output tokens were consumed for the specific answer.

### Analysis Steps

A section with collapsible panels showing the **LLM's thought process** — which tools and queries the bot used during incident analysis.

Each step contains:

| Element             | Description                                                          |
| ------------------- | -------------------------------------------------------------------- |
| **Phase**           | Stage name (e.g., `PromQL Query`, `Elasticsearch Search`)            |
| **Query Arguments** | Parameters passed to the tool (e.g., PromQL expression or DSL query) |
| **Response**        | Abbreviated payload returned by the tool                             |

Click on a step header to expand or collapse the details.

::: tip Reusing queries
If the bot executed a successful PromQL query, you can copy it from the analysis steps and use it to build a Grafana dashboard or configure alert rules.
:::

### Metadata Sidebar

To the right of the main content is a panel with incident metadata:

| Field             | Description                                   |
| ----------------- | --------------------------------------------- |
| **Duration**      | Time spent on analysis                        |
| **Detected**      | Date and time the incident was detected       |
| **Input Tokens**  | Number of tokens sent to the LLM              |
| **Output Tokens** | Number of tokens generated by the LLM         |
| **Bot**           | Name of the agent that processed the incident |
| **Incident ID**   | Unique identifier                             |
| **Severity**      | Severity level (`high` / `medium` / `low`)    |

## Billing & Tokens (`/dashboard/billing`)

SREBot uses a **token-based** billing system to charge for LLM invocations. All spending and top-up information is consolidated on this page.

### Current Balance

At the top of the page is your organization's **current token balance**. Below the balance is a **mini bar chart for the last 14 days**, where:

- **Green bars** represent top-ups (credits);
- **Red bars** represent usage charges (debits).

The chart lets you quickly assess spending dynamics.

### Current Tariff

Next to the balance is a card showing the **active tariff plan**:

| Parameter               | Description                                                                                  |
| ----------------------- | -------------------------------------------------------------------------------------------- |
| **Price per 1M tokens** | Cost per one million input tokens                                                            |
| **Output multiplier**   | Coefficient applied to the base price for output tokens (output is typically more expensive) |
| **Currency**            | Billing currency — **RUB** or **USD**                                                        |

::: info Currency selection
The organization's currency is determined **at creation time** based on the selected interface language and cannot be changed later.
:::

### Adding Funds (Top-up)

1. Click the **"Top Up"** button next to the balance.
2. In the modal dialog, enter the **number of tokens** to add.
   - Minimum: **100,000 tokens**
   - Maximum: **100,000,000 tokens**
3. The system automatically calculates the **estimated cost** in your organization's currency.
4. Click **"Pay"** — you will be redirected to the payment gateway.

After successful payment, the balance updates automatically.

### Transaction History

At the bottom of the page is a table of all balance operations with filtering support:

| Filter               | Description                    |
| -------------------- | ------------------------------ |
| **Transaction Type** | Filter by category (see below) |
| **Date Range**       | Filter by transaction date     |

#### Transaction Types

| Type       | Description                                        |
| ---------- | -------------------------------------------------- |
| **Usage**  | Charge for incident analysis or follow-up question |
| **Top-up** | Balance top-up via payment gateway                 |
| **Gift**   | Gift tokens credited by an administrator           |
| **Refund** | Refund (e.g., for an erroneous charge)             |

**Usage** transactions contain a link to the associated incident — click it to navigate to the incident detail page and see exactly what the tokens were charged for.

### Negative Balance Protection

::: danger Analysis blocked
If your balance drops **below zero**, SREBot **stops analyzing new alerts**. Instead, a notification is sent to the chat (Telegram/Slack/Discord) indicating the need to top up. Analysis resumes automatically after the balance is replenished.
:::

## Account & Settings (`/dashboard/settings`)

The settings page lets you manage your profile parameters and organization context.

### Changing Password

To change your password:

1. Enter your current password in the **"Current Password"** field.
2. Enter a new password in the **"New Password"** field (minimum 8 characters).
3. Confirm the new password in the **"Confirm Password"** field.
4. Click **"Save"**.

### Organization Context

The settings page displays information about the current organization:

- **Organization Name**
- **Your Role** in this organization (Owner / Admin / Member / Viewer)

To manage organization members, navigate to the **"Team"** section (available only to Owners and Admins). For more details, see [Team Management](/en/guide/team-management).
